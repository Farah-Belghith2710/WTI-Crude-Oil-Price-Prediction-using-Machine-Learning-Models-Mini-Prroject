2. Data Preprocessing

Data preprocessing transforms raw CSV data into a clean, model-ready format.

2.1 Date Conversion

Code:

df['date'] = pd.to_datetime(df['date'])

Purpose: - Converts date strings (e.g., "2015-01-02") to datetime objects - Enables time-based operations (sorting, filtering, resampling) - Allows feature extraction (year, month, day of week) - Creates proper time series visualizations

Why It Matters: Without datetime conversion, dates are just text strings. Machine learning models need proper temporal ordering to learn time-dependent patterns.

2.2 Numeric Conversion

Code:

numeric_columns = ['DJIA', 'NASDAQ', 'sp500', 'bitcoin', 'gold', 'Silver']
for col in numeric_columns:
    if df[col].dtype == 'object':
        df[col] = df[col].str.replace(',', '').astype(float)

Purpose: - Removes commas from formatted numbers (e.g., "25,000.50" → 25000.50) - Converts string data to numeric float type - Enables mathematical operations and model training

Example:

Before: "28,500" (string)
After:  28500.0 (number)

2.3 Index Setting

Code:

df.set_index('date', inplace=True)

Purpose: - Sets date column as DataFrame index - Optimizes time series operations - Enables easy date-based filtering: df['2020':'2022'] - Industry standard for time series data

2.4 Missing Value Handling

Code:

df.isnull().sum()  # Check missing values
df = df.fillna(method='ffill').fillna(method='bfill')

Missing Values Found: - Crude Oil (WTI): 64 missing values - Multiple variables: 53 missing values each

Filling Strategy: - Forward Fill (ffill): Uses last known value - Backward Fill (bfill): Uses next known value for remaining gaps

Rationale: For financial time series, forward fill is appropriate because prices don't change drastically overnight. This preserves data while making reasonable assumptions.

4. Feature Engineering

Feature engineering creates new variables from existing data to improve model predictions.

4.1 What is Feature Engineering?

Definition: Creating new, more useful variables from raw data to help models learn patterns better.

Analogy: - Raw data = Ingredients - Feature engineering = Cooking techniques - Model = Diner who benefits from preparation

4.2 Lag Features

Concept: Use past values to predict future

Code Example:

df['WTI_lag_1'] = df['crude oil ( WTI)'].shift(1)   # Yesterday
df['WTI_lag_7'] = df['crude oil ( WTI)'].shift(7)   # 7 days ago
df['WTI_lag_30'] = df['crude oil ( WTI)'].shift(30) # 30 days ago

Why Useful: - Oil prices are autocorrelated (today depends on yesterday) - Captures momentum: if oil was $80 yesterday, unlikely to be $50 today - Different lags capture different patterns (short-term vs long-term trends)

Example:

Date        WTI_Today  WTI_lag_1  WTI_lag_7
2020-01-01    $60        NaN        NaN
2020-01-02    $61        $60        NaN
2020-01-08    $62        $61        $60

4.3 Rolling Statistics (Moving Averages)

Concept: Calculate statistics over a sliding window

What is "Rolling"? A moving window that slides through your data, calculating statistics within that window.

Code Example:

# 7-day moving average
df['WTI_MA_7'] = df['crude oil ( WTI)'].rolling(window=7).mean()

# 30-day moving average
df['WTI_MA_30'] = df['crude oil ( WTI)'].rolling(window=30).mean()

# 7-day volatility
df['WTI_volatility_7'] = df['crude oil ( WTI)'].rolling(window=7).std()

How It Works:

Day 7 Calculation:

Window: Jan 1-7 → [60, 61.5, 59.8, 62.2, 61, 63.5, 62.8]
Average = (60 + 61.5 + 59.8 + 62.2 + 61 + 63.5 + 62.8) / 7 = $61.54

Day 8 Calculation:

Window: Jan 2-8 → [61.5, 59.8, 62.2, 61, 63.5, 62.8, 64]
Average = $62.11

The window "rolls" forward by one day!

Why Useful: - Smooths noise: Raw prices are volatile, moving averages show true trends - Trading signals: When short MA crosses long MA → trend change - Reduces overfitting: Models learn patterns, not noise

Visual Comparison:

Raw Prices:    *  *    *  * *   *  ← Noisy, hard to see trend
MA_7:          ___----____----___  ← Smooth, clear trend

4.4 Percentage Price Change

Code:

df_features['wti_price_change'] = df_features[target_col].pct_change()

Formula:

pct_change = (Today's Price - Yesterday's Price) / Yesterday's Price

Example:

Date          Price    pct_change    Interpretation
2020-01-01    $60      NaN          (no previous day)
2020-01-02    $63      +0.050       +5.0% increase
2020-01-03    $61      -0.032       -3.2% decrease
2020-01-04    $65      +0.066       +6.6% increase

Why Use Percentage Instead of Dollar Amount?

Problem with raw prices:

$60 → $63 = +$3 change = +5.0% (significant!)
$120 → $123 = +$3 change = +2.5% (less significant)

Benefits: 1. Scale independence: Fair comparison across price levels 2. Stationarity: Statistical properties consistent over time (critical for ML!) 3. Trader perspective: Investors think in % returns, not dollar changes

5. Feature Selection

5.1 Correlation-Based Filtering

Code:

correlations = X.corrwith(y).abs()
selected_features = correlations[correlations > 0.5].index.tolist()

Purpose: Keep only features with strong correlation to oil prices (> 0.5)

Why Filter Features?

Problems with too many features: - ❌ Some features add noise instead of signal - ❌ Slower training - ❌ Overfitting risk (curse of dimensionality) - ❌ Model gets confused

Benefits of filtering: - ✓ Faster training (50 → 6 features) - ✓ Better performance (less noise) - ✓ Easier interpretation

Correlation Values: - +1.0: Perfect positive relationship (X↑ → Y↑) - 0.0: No relationship - -1.0: Perfect negative relationship (X↑ → Y↓)

Threshold of 0.5: - Below 0.5 → Weak predictor, remove - Above 0.5 → Strong predictor, keep

6. Train-Test Split

6.1 The Fundamental Principle

You CANNOT test a model on the same data you trained it on!

Code:

split_index = int(len(X) * 0.8)
X_train, X_test = X[:split_index], X[split_index:]
y_train, y_test = y[:split_index], y[split_index:]

6.2 The 80/20 Split

With 2,569 days of data: - Training: First 2,055 days (80%) → 2015-01-01 to ~2022-08-15 - Testing: Last 514 days (20%) → ~2022-08-16 to 2024-12-31

Visual Timeline:

├──────────────────────────────────┼────────────┤
│   TRAINING SET (80%)             │  TEST (20%)│
│   2,055 days                     │  514 days  │
│   2015-01-01 → 2022-08-15        │  2022-08-16→│
└──────────────────────────────────┴────────────┘
     Train model here               Evaluate here

6.3 Why NOT Random Split for Time Series?

❌ WRONG (using sklearn's train_test_split):

# This randomly shuffles data - BAD for time series!
X_train, X_test = train_test_split(X, y, test_size=0.2)

Problem:

After random split:
Training: 2015-01-01, 2020-05-15, 2023-11-10...
Testing:  2016-03-11, 2022-07-04, 2015-06-30...

Issue: Testing on 2015 data while training on 2023 data!
→ FUTURE INFORMATION LEAKS INTO PAST! 🚨

✓ CORRECT (sequential split):

X_train, X_test = X[:split_index], X[split_index:]

Timeline preserved:
Training: 2015 → 2022 (past only)
Testing:  2022 → 2024 (future only)

Simulates reality: Predict future using only past! ✓

6.4 Why This Matters

Real Trading Scenario:

Today: December 13, 2024
You have data from: 2015 → Today
You want to predict: Tomorrow (Dec 14, 2024)

Your model:
✓ Trained on: 2015 → 2022
✓ Tested on: 2022 → 2024
✓ Now predicts: Dec 14, 2024

This is realistic! You can ONLY use past to predict future.

7. Feature Scaling

7.1 The Problem: Different Feature Scales

Before Scaling:

Feature          Min        Max        Range
Heating Oil      $0.56      $5.15      $4.59
gold             $1,052     $2,786     $1,734
bitcoin          $3,229     $106,157   $102,928  🚨 HUGE!
Natural GAZ      $1.54      $9.65      $8.11

Why is this a problem?

Machine learning algorithms calculate distances:

Distance between two points:
A: [Gold=$1,500, Bitcoin=$30,000, Oil=$60]
B: [Gold=$1,510, Bitcoin=$35,000, Oil=$62]

Euclidean Distance = √[(10)² + (5,000)² + (2)²]
                   = √[100 + 25,000,000 + 4]
                   ≈ 5,000

Bitcoin dominates! Gold and Oil barely matter!

7.2 StandardScaler Solution

Code:

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)    # Learn & apply
X_test_scaled = scaler.transform(X_test)          # Apply only

What StandardScaler Does: Transforms each feature to have: - Mean = 0 - Standard Deviation = 1

Formula:

scaled_value = (original_value - mean) / standard_deviation

Example with Gold:

Original Gold prices: [1500, 1520, 1480, 1510, 1505]
Mean = 1503, Std = 14.7

Scaled values:
1500 → (1500 - 1503) / 14.7 = -0.20
1520 → (1520 - 1503) / 14.7 = +1.16
1480 → (1480 - 1503) / 14.7 = -1.56
1510 → (1510 - 1503) / 14.7 = +0.48

7.3 Critical Rule: fit vs transform

✓ CORRECT:

scaler.fit_transform(X_train)  # Learn stats from training data
scaler.transform(X_test)       # Apply SAME stats to test data

❌ WRONG:

scaler.fit_transform(X_train)  # Learn from train
scaler.fit_transform(X_test)   # Learn from test → DATA LEAKAGE!

Why? Training and test must use the SAME scaling parameters for fair comparison!

7.4 Before vs After Scaling

Before:

[Gold=$1,500, Bitcoin=$30,000, Oil=$60, HeatingOil=$2.50]
→ Bitcoin dominates due to large numbers!

After:

[Gold=-0.20, Bitcoin=-0.15, Oil=-0.18, HeatingOil=-0.22]
→ All features on similar scale! Model treats them equally ✓

8. Model Training

We trained 5 different regression models to compare performance.

8.1 Linear Regression

Code:

lr = LinearRegression()
lr.fit(X_train_scaled, y_train)
predictions['Linear Regression'] = lr.predict(X_test_scaled)

How It Works: Finds the best straight line (hyperplane) through data.

Equation:

WTI_Price = β₀ + β₁×Feature₁ + β₂×Feature₂ + ... + βₙ×Featureₙ

Strengths: - ✓ Simple and fast - ✓ Highly interpretable - ✓ No hyperparameters to tune

Weaknesses: - ❌ Assumes linear relationships - ❌ Sensitive to outliers - ❌ Can't capture complex patterns

8.2 Random Forest Regressor

Code:

rf = RandomForestRegressor(
    n_estimators=100,    # Number of trees
    max_depth=15,        # Tree depth
    random_state=42,
    n_jobs=-1           # Use all CPU cores
)
rf.fit(X_train_scaled, y_train)

How It Works: Creates 100 decision trees and averages their predictions.

Process: 1. Randomly sample training data 2. Build a decision tree 3. Repeat 100 times 4. Average all predictions

Strengths: - ✓ Robust to outliers - ✓ Captures non-linear patterns - ✓ Less overfitting than single tree

Weaknesses: - ❌ Slower than Linear Regression - ❌ Less interpretable ("black box")

8.3 XGBoost (Extreme Gradient Boosting)

Code:

xgb_model = xgb.XGBRegressor(
    n_estimators=100,
    learning_rate=0.1,
    max_depth=6,
    random_state=42,
    n_jobs=-1
)
xgb_model.fit(X_train_scaled, y_train)

How It Works: Builds trees sequentially, each correcting errors of previous ones.

Process: 1. Tree 1 predicts prices 2. Calculate errors 3. Tree 2 learns to predict those errors 4. Combine: Final = Tree1 + (0.1 × Tree2) 5. Repeat 100 times

Strengths: - ✓ Often the BEST performing model - ✓ Fast training despite complexity - ✓ Built-in regularization

Weaknesses: - ❌ Requires parameter tuning - ❌ Can overfit if not careful

8.4 Gradient Boosting Regressor

Code:

gb = GradientBoostingRegressor(
    n_estimators=100,
    learning_rate=0.1,
    max_depth=5,
    random_state=42
)
gb.fit(X_train_scaled, y_train)

How It Works: Very similar to XGBoost (original implementation of gradient boosting)

XGBoost vs Gradient Boosting: - XGBoost = Optimized C++ version (faster) - Gradient Boosting = Python sklearn version (slower but stable)

8.5 Support Vector Regressor (SVR)

Code:

svr_model = SVR(
    kernel='linear',
    C=100,
    gamma='auto',
    epsilon=0.1
)
svr_model.fit(X_train_scaled, y_train)

How It Works: Finds a "tube" around data where most points fit, ignoring outliers.

Parameters: - kernel='linear': No transformation (simple line) - C=100: Strict fitting (high penalty for errors) - epsilon=0.1: Tolerance margin (±$0.10)

Strengths: - ✓ Robust to outliers - ✓ Memory efficient

Weaknesses: - ❌ Slow on large datasets - ❌ Sensitive to parameters

9. Evaluation Metrics

We calculated 4 metrics to comprehensively evaluate each model.

9.1 RMSE (Root Mean Squared Error)

Formula:

RMSE = √[(1/n) × Σ(Actual - Predicted)²]

What It Measures: Average prediction error in dollars

Example:

Actual:    [$60, $65, $58, $70]
Predicted: [$62, $63, $59, $68]
Errors:    [-$2, +$2, -$1, +$2]
Squared:   [$4, $4, $1, $4]
RMSE:      √(13/4) = $1.80

Interpretation: - RMSE = $1.80 → Predictions off by $1.80 on average - Lower is better (0 = perfect)

Key Feature: Penalizes large errors more (due to squaring)

9.2 MAE (Mean Absolute Error)

Formula:

MAE = (1/n) × Σ|Actual - Predicted|

What It Measures: Average prediction error without squaring

Example:

Actual:    [$60, $65, $58, $70]
Predicted: [$62, $63, $59, $68]
Errors:    [-$2, +$2, -$1, +$2]
Absolute:  [$2, $2, $1, $2]
MAE:       (2+2+1+2)/4 = $1.75

Interpretation: - MAE = $1.75 → Typical error is $1.75 - More intuitive than RMSE

Key Feature: Treats all errors equally (no extra penalty for large errors)

9.3 MAPE (Mean Absolute Percentage Error)

Formula:

MAPE = (1/n) × Σ|(Actual - Predicted) / Actual| × 100%

What It Measures: Average error as percentage of actual value

Example:

Actual:    [$60,  $65,  $58,  $70]
Predicted: [$62,  $63,  $59,  $68]
% Errors:  [3.3%, 3.1%, 1.7%, 2.9%]
MAPE:      (3.3+3.1+1.7+2.9)/4 = 2.75%

Interpretation: - MAPE = 2.75% → Predictions off by 2.75% - Scale-independent (can compare across datasets)

Key Feature: Shows relative error (more meaningful than absolute error)

9.4 R² Score (Coefficient of Determination)

Formula:

R² = 1 - (Model_Error / Baseline_Error)

What It Measures: Percentage of variance explained by model

Example:

Baseline (always predict mean): Error = 86.75
Your model: Error = 13
R² = 1 - (13/86.75) = 0.85 = 85%

Interpretation: - R² = 0.85 → Model explains 85% of variance - Range: -∞ to 1.0 - 1.0 = Perfect predictions - 0.0 = No better than predicting mean - Negative = Worse than predicting mean

R² Benchmarks:

> 0.9   = Excellent
0.7-0.9 = Good
0.5-0.7 = Moderate
< 0.5   = Poor

Appendix B: Glossary

Autocorrelation: Correlation of a time series with its own past values

Correlation: Statistical measure of relationship between two variables (-1 to +1)

Feature: Input variable used by model for predictions

Feature Engineering: Creating new variables from existing data

Hyperparameter: Model setting that must be specified before training

Lag Feature: Past value of a variable used as predictor

Overfitting: Model memorizes training data but fails on new data

R² (R-Squared): Proportion of variance explained by model (0 to 1)

RMSE: Root Mean Squared Error - average prediction error

Scaling: Transforming features to similar ranges

Stationarity: Statistical properties remain constant over time

Train-Test Split: Dividing data into training and evaluation sets
